## Demo Video Script

Length: 45-60 seconds.

Structure:

1. 0-5s: open with the play dashboard.
   Voiceover: `This is 5e Character Vault, a local-first character sheet for DnD 5e.`

2. 5-15s: show HP, damage, healing, XP, resources.
   Voiceover: `The play mode keeps table-critical controls large and close: HP, resources, XP, death saves, and conditions.`

3. 15-25s: open dice modal and roll.
   Voiceover: `Dice are built in, including physical 3D rolls for common 5e dice.`

4. 25-38s: show edit mode, spellbook, inventory.
   Voiceover: `The full sheet is still there: identity, combat, inventory, coins, resources, spells, and notes.`

5. 38-50s: open floating history and export.
   Voiceover: `Everything stays in the browser with autosave history and portable JSON backups.`

6. 50-60s: close on URL.
   Voiceover: `It is free, open source, and ready to try.`

## Launch Content Plan

### 7-5 Days Before Launch

- Create Product Hunt maker profile with real name, photo, and GitHub/social links.
- Prepare all gallery images and test the first image at 240x180 crop.
- Record the demo video and upload it to YouTube as unlisted.
- Ask 5-10 DnD players or GMs to test the live URL and report blockers.
- Make sure the GitHub README explains local-first storage and backup.

### 3 Days Before Launch

- Create a Product Hunt draft with all copy and visuals.
- Verify the product URL works in a clean browser profile.
- Prepare personal outreach messages for friends, makers, DnD groups, and open-source contacts.
- Write a short changelog post on GitHub or social.

### 1 Day Before Launch

- Final screenshot pass on desktop and mobile.
- Confirm links: live app, GitHub source, license, author.
- Prepare launch posts for X, LinkedIn, Reddit/Discord communities where self-promotion is allowed.
- Sleep; launch day is mostly replies.

### Launch Day

- Publish at 12:01 AM Pacific if scheduling on Product Hunt.
- Post the maker first comment immediately.
- Share the launch with context, not just a bare link.
- Reply to every comment with substance.
- Track bugs in GitHub issues and ship small fixes quickly.

### Day After

- Thank supporters.
- Publish a short results/learning post.
- Convert repeated feedback into GitHub issues.
- Add Product Hunt badge only if the launch gets meaningful traction.

## Positioning

Primary angle:

`A practical DnD 5e sheet for live play, not another account-gated character database.`

Differentiators:

- Local-first: character data stays in the browser.
- No account required.
- Built-in dice and session controls.
- Autosave history and manual JSON backups.
- Open source under MIT.
- Desktop and mobile friendly.

Potential objections:

- `Is this official D&D Beyond?`
  No. It is an independent, open-source 5e utility.

- `Will my character sync across devices?`
  Not yet. The current model is local-first with JSON export/import.

- `Will data survive browser cleanup?`
  Browser storage can be cleared by the user or browser. Export JSON backups for safety.
